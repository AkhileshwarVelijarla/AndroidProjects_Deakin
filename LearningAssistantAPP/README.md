# LLM-Enhanced Learning Assistant App

Task name: Task 6.1D LLM-Enhanced Learning Assistant App

This Android app is a tutor-friendly student learning assistant built with Java and MVVM. It uses dummy learning data plus mock-capable LLM utilities to demonstrate personalised learning, adaptive assessment, and intelligent tutoring flows without requiring a real API key.

## Features

- Dummy login flow with welcome screen and navigation to account setup
- Account registration with empty-field validation, email confirmation, and password matching
- Multi-select interests screen using locally stored preferences
- Home dashboard with greeting, selected interests, dummy quiz history, and generated learning tasks
- AI utility for question hints
- AI utility for answer explanations
- AI utility for 7-day study plans
- AI utility for flashcard generation
- Prompt, response, loading, error, and retry UI for every LLM action
- Quiz assessment with multiple-choice questions and score calculation
- Results screen with feedback per question and AI follow-up study planning
- SharedPreferences persistence for user profile, interests, and latest quiz result

## LLM Utilities Used

- Create 3 flashcards from a selected interest topic

## Mock LLM Mode

The app runs fully without any API key.

- If `LLM_API_KEY` is missing from `local.properties`, `BuildConfig.MOCK_LLM` becomes `true`
- In mock mode, `LLMApiService` returns realistic dummy AI responses after a short loading delay
- Each AI area also includes a `Simulate API failure` checkbox so the marker can test the error and retry UI

## Where To Add The API Key

Add these entries to `local.properties`:

```properties
LLM_API_KEY=your_real_api_key_here
LLM_BASE_URL=https://your-llm-endpoint.example.com
```

Notes:

- `app/build.gradle.kts` reads these values and exposes them through `BuildConfig`
- `LLMApiService.java` contains a comment showing where a real network call should be inserted
- No real API keys are hardcoded in the committed source

## How To Run

1. Open the project in Android Studio.
2. Let Gradle sync.
3. Run the `app` configuration on an emulator or Android device.
4. Login with any non-empty username and password, or create a new account.
5. Select interests, open the dashboard, start the assessment, and test the AI utilities.
